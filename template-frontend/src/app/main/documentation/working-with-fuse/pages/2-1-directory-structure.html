Working with Fuse Fuse
Directory Structure
Demo Structure

Here you can see the directory structure of the Fuse:

Note: This directory structure is from the demo application. Skeleton project may have a slightly different structure.

app/core

Contains all the core elements that required for Fuse to work. It's highly recommended NOT to touch or modify anything inside the core directory so you can easily update the template when a new version released.

app/data

Contains all the json files that holds demo data. We are using json files to emulate API calls and the data flow. These json files are good for seeing the data structure we are using in our demo applications so you can use similar data structure to minimize your work.

app/main

Contains all the modules from theme demo. Apps, Components, Custom Pages and everything else you can see in the theme demo are located in this directory.

app/navigation

Contains the main navigation sidenav.

app/quick-panel

Contains the extra sidenav called Quick Panel.

app/toolbar

Contains the main toolbar module. It's separated from the core as the same reason with the sidenav modules.

index.***.js

These files are the entry point of the main (fuse) application. Fuse is an Angular module by itself. It means you can either create your app on top of the Fuse, or you can add Fuse as another module into your current project. Keep in mind, adding Fuse into your own app as a module might require some extra work.

index.scss

This is the main scss file. DO NOT REMOVE it as all the other scss files will be injected into this file before compiled to a css file.
