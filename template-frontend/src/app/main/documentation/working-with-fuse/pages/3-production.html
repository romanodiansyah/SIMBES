Building? What is that?

As we mentioned earlier, Fuse is more than a traditional admin template. It's more of a base application that you can start working with in order to speed up your development process. Once you finish your development, you can build the application. Building process is a combination of couple other processes.

Mainly, after building your application, all your html files are going to be injected into a single javascript file and they will become entities in the Angular's $templateCache service. All your style and javascript files will be minified and uglified. And most importantly, you will be able to run your application from a simple host without needing Node.js, Gulp or any other tools we mentioned.

Your app will become a simple HTML/Javascript application.

Building

To build your application, simply run the following command in your favorite console application (Terminal, Command Prompt etc.) and wait for it to finish: gulp build

And that's it. Now the production version of your app is living under the /dist directory within your work folder. You can directly upload the contents of /dist folder to your server, and it will work.

.htaccess

Fuse uses HTML5 mode in its location service. It means, you will see URLs like www.example.com/mail rather than seeing URLs like this one: www.example.com/#/mail

Also, without following .htaccess rules, you can't access your application directly. So typing www.example.com/mail to the url bar or refreshing your application will cause an Internal Server Error.

Create an .htaccess that stays with the same directory as your index.html and paste the following code inside it: 

<IfModule mod_rewrite.c>
    RewriteEngine On
    # Send all requests to the index.html unless
    # it's a directory or a file that actually exists
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteRule ^ index.html [L]
</IfModule>
