Server

As we mentioned before, Fuse uses Yeoman generator for AngularJS with GulpJS for quickly bootstrapping the application and it comes with a built-in server setup so you don't have to worry about setting up a server for your development environment.

The built-in server setup uses browser-sync. There are a lot of benefits of using browser-sync such as testing your app in multiple browser windows and even across multiple devices. Browser-sync will synchronise your actions such as clicking and scrolling across browser windows and devices to make testing easier.

To start the server, run the following command in your favorite console application (Terminal, Command Prompt etc.) while in your work folder: gulp serve

This will start up the server, compile scss files, inject dependencies, and will do all other related jobs for you. Then it will automatically fire up your default browser and redirect you to the http://localhost:3000 by default.

You can check out your console application to get further information about the server. By default, it will run on http://localhost:3000 but it might change depending on your setup.

Reloading the Server

The gulp serve command will watch your files and reload the page for you as you make changes. But for some reason, if you need to manually restart the server, you can do it by pressing Ctrl + C on your keyboard while in the console application and then run the following command once again: gulp serve
